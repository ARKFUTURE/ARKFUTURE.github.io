# 公约 [v3.1 UTF-8]


## 简介: 
* ARKFUTURE 成立于 2015 年, 由一群对技术充满热情的爱好者共同组建的集成型工作室(团队).经过多年的稳步发展, 我们已从最初专注于游戏制作的兴趣小组, 逐步成长为融合多种技术于一体的综合性技术团队.
* 我们聚集一堂, 交流观点与创意, 共同促进进步; 深入研究, 共同应对挑战; 彼此协作并分享经验, 持续探索未知,突破自我.
* 我们不特别注重您的教育背景和以往经历, 只要您对学习抱有热忱, 渴望不断丰富知识库与兴趣技能, 且具备持续学习的坚定信念, 便可以成为我们团队的一员.
* 核心定位: 以 INGO(国际非政府组织),INPO(国际非营利组织)为基础, 践行 OS-GNU(开源共创及 GNU 精神)与 CL(持续学习)理念, 构建无边界的技术交流生态.


## 理念 / 性质 / 使命
* 工作室理念: 技术无止, 勇于创新; 以史为镜, 洞察未来

# 第一章 总则
## 第一章第一节 核心规则
1. 第一条 ARKFUTURE(以下简称 "社区")旨在为技术爱好者提供一个自由交流,协作学习的平台.为鼓励良性互动, 促进共建共享, 规范社区秩序, 特修订本公约.社区对成员的学历和年龄不设限制, 只要对学习和技术抱有热情, 愿意持续提升个人知识储备与技能素养, 并持有长期学习的态度, 均可以成为我们的一员; 社区成员的学习动机应出于个人爱好.
2. 第二条 成员加入 ARKFUTURE 后, 必须阅读并遵守本公约.公约以 "监督/保障/自由" 为核心, 所有条款均为维护社区基本秩序, 同时确保成员拥有自由学习,共享技术资源,使用技术资源的合法权利.
3. 第三条 ARKFUTURE 致力于构建一个具备开放性,包容性,自由性,共享性和互动性的技术社区, 倡导在 "文明,互助,协作,共赢" 的基础上: 
```
   1. 彼此尊重, 平等沟通; 
   2. 相互学习, 彼此协助; 
   3. 协同合作, 共同提升; 
   4. 彼此分享, 共同发展; 
   5. 理性表达, 友好交流; 
   6. 真诚友善参与社区建设, 不以一己私利为重, 共建良性生态.
```
4. 第四条 我们倡导匿名,平等,言论自由,分布式自我监管的技术社区, 不涉及任何不道德,不健康的内容.管理运维工作组的存在仅为 "保障自由而非限制自由", 仅在违反公约核心规则时介入, 不干预成员的技术交流与合法观点表达.
5. 第五条 我们倡导技术的自由性(GNU/FSF), 反对任何形式的不合理限制或束缚, 坚信开放交流方能实现共赢(封闭仅会催生竞争者的进步); 版权归属于原创者及参与迭代合作的作者.鼓励成员分享代码,优化他人作品, 尊重不同价值观, 通过建设性对话寻求 "自由发展与权益保护" 的平衡点.
6. 第六条 禁止对成员的劳动成果进行嘲笑,侮辱,诋毁或攻击; 尊重并包容他人成果, 即便进步微小或未见明显进展, 亦应予以肯定和鼓励.
7. 第七条 成员需持续学习各类技术充实自身, 并在各领域中相互交流协助.
8. 第八条 严禁泄露 ARKFUTURE STUDIO (GROUP) 及资源创作者的任何信息与未授权资源(包括内部讨论记录,未开源代码,成员私有分享资源等), 除非获得相关主体的书面授权.禁止在公共场合及公开渠道发布未授权资源.
9.  第九条 团队成员与管理运维人员应恪守中立原则, 不参与任何形式的政治等站队行为.
10.  第十条 本公约自社区成立当年起生效并立即执行, 修订需经管理运维工作组集体协商后公示.

## 第一章第二节 补充规则
1. 禁止傲慢心态, 保持虚心学习,共同创造,独立思考的社区氛围; 技术水平高低不影响平等地位, 不欢迎傲慢自大者加入.
2. 禁止破坏社区公共服务资源(包括但不限于 DDoS 攻击,洪水攻击,发布垃圾广告等), 此类行为不受欢迎, 鼓励成员举报恶意行为.
3. 减少使用 AI 生成内容发言(发帖), 社区无积分 / 经验值系统, 无需为灌水使用 AI 生成内容; 技术,新闻等分享需基于现实实际, 核心为个人技术交流与讨论.

# 第三章 成员・成员节点

4. 拒绝骗子及恶意广告发布者加入, 收到成员举报后, 管理运维人员应立即核实处理.

## 第三章第二节 成员节点
1. 网络节点(Network Node)指计算机网络中的关键组成部分(如计算机,服务器,路由器等), 具有唯一标识符及特定位置与功能.
2. 社区引入 "成员节点(Member Node)" 概念, 指在分布式资源共享与协作社区中, 具备独立技术(技能), 并参与技术经验分享,资源交换与协作的成员; 每个成员节点既是资源贡献者(Provider), 也是资源消费者(Consumer), 形成去中心化协作生态.
3. 引入 "成员节点拓扑(Member Node Topology)" 概念, 描述成员节点间的关系方式, 网络拓扑中节点间的关系在此同样适用.
4. 每个成员均可作为独立节点, 通过分享技术,资源和知识与其他成员建立联系, 实现互相学习,协作共赢.
5. 成为成员节点的要求: 
```
### 1. 个人意愿确认(必需)
* 拥有独立的技术(技能)或资源; 
* 愿意分享自身技术(技能)或资源; 
* 愿意参与社区的学习和协作; 
* 遵守社区的规定和规则; 
* 愿意为社区的发展做出贡献.

### 2. 个人条件(非必需)
* 主要成员节点(作为服务): 需具备 I2P,Tor,GnuNet 等区中心或匿名网络环境; 
* 普通成员节点: 建议具备 BitTorrent,IPFS 等匿名文件共享网络, IRC,XMPP,Matrix,Telegram 等匿名聊天平台, Git 搭建的个人知识库或任意开源知识库(其他相关工具可补充).
```
6. 技术价值总结: 
```
资源及知识垄断 → 转变为分布式知识民主化, 共同协作共赢化
被动消费, 参与不积极 → 升级为主动参与式学习, 个人需要式学习
数据孤岛 → 演化为跨节点价值流动网络, 知识技术共创共享
```


# 第四章 管理运维人员
## 第四章第一节 核心定义
1. **管理运维工作组**由 ARKFUTURE 成员自发组成, 成立初衷仅为维护社区秩序, 不具备行政强制权.
2. 所有管理运维成员需遵守中立管理原则, 不参与任何形式的政治等站队行为.
3. 管理运维工作组的存在是为了 [保障自由] 而非 [限制自由], 仅在违反公约核心规则(如恶意破坏、侵权、歧视)时介入, 不干预成员的技术交流,观点表达.
4. 管理职位由自愿负责监督的成员担任, 采用独一无二的用户名至关重要: 

> 将要求提供**用户名**并确保提交的 **用户名**具有唯一性和专属性, 以防止任何形式的滥用或混淆
> 
> 且在ARKFUTURE-NET所有服务中使用的**用户名(UserName)/昵称(NickName)**跟提交的**用户名**是相同的 
>
> 为了保证个人隐私, **用户名** 不必是社交媒体账号或个人常用标识, 其仅用于在 社区 中识别管理员身份

## 第四章第二节 公钥管理规则
1. 申请担任管理人员时, 需使用个人用户名及任意邮箱生成 OpenPGP 密钥对, 将公钥导出提交至 [keys.openpgp.org](https://keys.openpgp.org), 并向管理运维工作组提交公钥 ID 以便公示.
2. 密钥对有效期需为 90day-360day(推荐 180 天)的短期密钥, 永久时效密钥对不予认可; 到期需及时续期, 密钥对,公钥或公钥 ID 有变化时, 需及时联系 arkfuture 告知变更.
3. 普通成员可通过公示的公钥 ID 查询 [keys.openpgp.org](https://keys.openpgp.org) 中的公钥信息, 验证管理员身份(如验证管理员签署的签名信息,解密管理员加密的资源等).

# 第五章 知识产权及版权
1. 资源发布者在 ARKFUTURE 上发布的原创内容, 著作权归原作者本人所有; 为促进知识分享与传播, 发布者授予 ARKFUTURE 全球通用,永久,免费,不可撤销,可再许可,可转让的非独家许可使用权(该许可仅用于社区内知识共享, 管理团队不得用于商业用途).

3. 发布者提出的关于 ARKFUTURE 服务的想法,建议和反馈, 社区可无偿使用, 无需向发布者补偿或承担义务.
4. 除特别说明或法律另有规定外, ARKFUTURE 公开的全部信息或数据(包括但不限于音频,视频,文字,图片,图形,网站架构,程序设计,程序代码,页面设计,交互生成的元数据等)的其他权益归 ARKFUTURE 所有.


# 第六章 举报与授权流程
## 第六章第一节 举报流程
1. 成员发现违规行为, 可通过 IRC 私信,Telegram 私信联系管理员账号(arkfuture), 并提供相关证据(如聊天记录截图,资源链接等).
2. 管理员需在 48 小时内响应核实, 核实后根据违规情节采取提醒,警告,移除权限等措施, 并将处理结果反馈举报者(匿名举报可仅公示处理结果).
## 第六章第二节 资源授权流程
1. 成员需对外分享社区资源时, 需向资源原创者及管理员提交书面授权申请(可通过 PGP 加密发送).
2. 授权通过后, 可标注 "ARKFUTURE 授权分享" 对外发布, 未获授权不得私自传播.
